--------------------------------------------------
ETAPA 2 — VERIFICATION (VALIDAÇÃO TÉCNICA)
--------------------------------------------------

VALIDAR:

AUTH:
- Sessão não é invalidada indevidamente
- SESSION_INVALIDATED ocorre apenas quando necessário
- DEVICE_MISMATCH está correto
- login/logout funcionando sem loop

PERMISSÕES:
- FULL_ACCESS_ROLES funcionando
- MASTER/ADMIN/DIRECTOR têm acesso total
- usuários restritos NÃO são redirecionados → recebem "Acesso negado"

FRONTEND:
- fetchWithAuth NÃO dispara logout em 401 não críticos
- IGNORE_401_URLS funcionando
- erro de permissão NÃO redireciona para /login

BANCO:
- conexão usando SUPABASE_DATABASE_URL
- SSL ativo
- sem fallback indevido para banco local
- migrations aplicadas

CACHE:
- queries com refetch correto após mutation
- sem stale infinito problemático
- UI atualiza sem reload

SEGURANÇA:
- senhas com hash (bcrypt/argon2)
- nenhuma senha hardcoded
- secrets NÃO expostos no código
- validação de input básica
