🎯 FASE 8.6J — ISOLAMENTO MULTI-TENANT (OUTBOX WORKER)

Adicionar isolamento por tenant no worker de eventos (startOutboxWorker), garantindo que cada evento seja processado dentro do contexto correto da empresa.

---

🚫 REGRAS CRÍTICAS

1. NÃO alterar lógica de negócio
2. NÃO alterar payload dos eventos
3. NÃO alterar estrutura de logs
4. NÃO remover código existente
5. NÃO alterar queries SQL
6. NÃO alterar retries ou controle de fila

👉 Apenas adicionar isolamento com runWithTenant por evento

---

🧠 CONTEXTO

- Worker atual processa eventos de múltiplos tenants em loop único
- payload já contém companyId
- NÃO existe contexto de tenant ativo (ALS)
- risco atual: chamadas internas podem operar sem tenant

📦 TAREFAS

## 1️⃣ IMPORTAR runWithTenant

## 2️⃣ WRAP POR EVENTO

Localizar loop principal:

ts for (const event of events) { 

---

### Substituir conteúdo interno por:

ts for (const event of events) {   const companyId = (event as any)?.payload?.companyId;    // fail-safe — não processa evento sem tenant   if (!companyId) {     console.warn("[OUTBOX] Evento sem companyId — ignorado por segurança");     continue;   }    const tenantPrincipal: TenantPrincipal = {     kind: "admin",     empresaId: companyId,     userId: 0,     role: "SERVICE",   };    await runWithTenant(     { principal: tenantPrincipal, empresaId: companyId },     async () => {       await dispatchEvent(event);     },   ); }
