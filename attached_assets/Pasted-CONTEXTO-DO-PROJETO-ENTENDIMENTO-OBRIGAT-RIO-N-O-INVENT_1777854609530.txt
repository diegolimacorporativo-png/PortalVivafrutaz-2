NUNCA:

2. CAMADAS DE DADOS

3. SEGURANÇA

- outros roles são sempre escopo de empresa

4. BANCO DE DADOS
- Drizzle ORM usado
- filtros devem ser SQL (não filter() em memória)
- in-memory filtering é considerado RISCO, exceto BI explícito
