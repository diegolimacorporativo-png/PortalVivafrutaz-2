OBJETIVO

Implementar sistema completo de assinaturas SaaS com controle de planos, módulos liberados por plano, métodos de pagamento (PIX, cartão e boleto), upgrade de plano, bloqueio automático por inadimplência, plano free configurável, integração com clientes VivaFrutaz e sistema de atualizações visível no guia rápido.

O sistema deve permitir que empresas utilizem o ERP conforme o plano contratado e que o administrador controle totalmente funcionalidades, pagamentos e atualizações.

------------------------------------------------

ARQUITETURA DE PLANOS

Criar tabela:

planos_saas

Campos:

id  
nome_plano  
descricao  
valor_mensal

tipo_plano (free, premium)  
limite_usuarios  
limite_pedidos_mes

limite_rotas  
gps_habilitado  
logistica_inteligente  
ativo  

------------------------------------------------

PLANO FREE

O plano free deve ser configurável pelo administrador.

O administrador pode escolher:

quais módulos estarão disponíveis  
quantidade de usuários  
limite de pedidos  

Exemplo:

Plano Free

Dashboard  
Clientes  
Pedidos básicos  

Sem:

GPS  
Logística inteligente  
IA

Se o módulo não estiver liberado o sistema deve esconder a aba.

------------------------------------------------

ASSINATURAS NA ABA CLIENTES

Permitir que clientes já cadastrados no sistema VivaFrutaz possam receber uma assinatura.

Ou seja:

um cliente pode virar uma empresa SaaS.

Ao ativar assinatura para um cliente ele ganha acesso ao sistema conforme o plano.

------------------------------------------------

MÉTODOS DE PAGAMENTO

Criar opções de pagamento:

PIX  
Cartão  
Boleto  

Campo:

metodo_pagamento

------------------------------------------------

PIX

Se pagamento for PIX:

gerar QR Code  
liberar acesso após confirmação.

------------------------------------------------

CARTÃO

Se pagamento for cartão:

autorizar pagamento  
ativar assinatura automaticamente.

------------------------------------------------

BOLETO

Se pagamento for boleto:

gerar boleto com data de vencimento  
liberar acesso temporário até vencimento.

Campos adicionais:

linha_digitavel  
data_vencimento  

------------------------------------------------

VERIFICAÇÃO DE BOLETO

Criar rotina automática diária:

checkBoletos()

Essa função deve:

verificar boletos vencidos  
verificar se pagamento foi confirmado  

Se boleto estiver vencido e não pago:

alterar status da assinatura para atrasada.

------------------------------------------------

BLOQUEIO AUTOMÁTICO

Se assinatura estiver atrasada:

bloquear acesso ao plano pago  
mover empresa automaticamente para plano free.

O sistema deve mostrar aviso:

"Seu plano expirou. Regularize pagamento para reativar funcionalidades."

------------------------------------------------

NOTIFICAÇÃO DE ATRASO

Enviar alerta para administrador quando uma assinatura estiver vencida.

Mensagem:

"Assinatura da empresa X está vencida."

------------------------------------------------

UPGRADE DE PLANO

Na área de pagamentos criar botão:

UPGRADE DE PLANO

Ao clicar o cliente pode:

selecionar novo plano  
selecionar forma de pagamento.

O sistema deve atualizar imediatamente as permissões do plano.

------------------------------------------------

DOWNGRADE AUTOMÁTICO

Se pagamento não for realizado após vencimento:

migrar empresa automaticamente para plano free.

------------------------------------------------

INTEGRAÇÃO COM LOGÍSTICA E GPS

Se plano possuir módulo GPS ou logística inteligente:

ativar automaticamente essas funcionalidades no ERP da empresa.

Caso contrário esconder essas abas.

------------------------------------------------
