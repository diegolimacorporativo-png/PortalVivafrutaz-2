🎯 OBJETIVO

Executar uma auditoria completa de segurança e produção de um SaaS multi-tenant, identificando:

* vazamentos de dados entre tenants
* falhas de autenticação/autorização
* billing falso ou incompleto
* NF-e insegura ou simulada
* limites apenas visuais
* cron jobs inativos
* IA com acesso indevido
* risco real de produção

⸻

📁 CONTEXTO DO SISTEMA

O sistema contém:

* arquitetura SaaS multi-tenant
* autenticação com middleware (ex: requireAuthCore)
* billing + webhooks
* NF-e (integração fiscal)
* módulos com limites por plano
* IA (agent interno)
* cron jobs
* frontend + backend + DB

🔍 INSTRUÇÕES DE EXECUÇÃO (OBRIGATÓRIO)

PASSO 1 — MAPEAMENTO DO SISTEMA

Antes de concluir qualquer análise:

* encontre middleware de auth
* encontre camada de tenant (resolveTenant / tenantId)
* identifique rotas críticas
* identifique billing + webhooks
* identifique NF-e flow

PASSO 2 — ANÁLISE POR DOMÍNIO

1. MULTI-TENANT (CRÍTICO)

Verificar em TODAS as queries:

* tenantId está sempre presente?
* existe isolamento real de dados?
* existe qualquer query sem filtro de tenant?

Classificar:

* SAFE
* RISK
* CRITICAL (vazamento entre empresas)

⸻

2. AUTH / AUTHORIZATION

* middleware aplicado em todas rotas?
* rotas públicas indevidas?
* bypass possível?

Classificar:

* SAFE / RISK / CRITICAL

⸻

3. BILLING

* gateway real ou mock?
* webhook validado (assinatura)?
* eventos persistidos?

Classificar:

* REAL / PARCIAL / MOCK

⸻

4. LIMITES (PLANO)

Verificar:

* enforcement backend OU apenas frontend?

Classificar:

* ENFORCED
* VISUAL ONLY

⸻

5. NF-e (FISCAL)

* emissão real ou simulada?
* integração SEFAZ ou provider?
* retry + erro tratado?

Classificar:

* SAFE / RISK / CRITICAL

⸻

6. CRON JOBS

* existem jobs registrados?
* estão rodando de fato?
* logs existem?

Classificar:

* ACTIVE / UNCERTAIN / INACTIVE

⸻

7. IA (AGENT SYSTEM)

* acessa banco?
* respeita tenant?
* pode executar ações perigosas?

⸻

8. FRONTEND SECURITY

* fetchWithAuth consistente?
* tratamento 401/403 centralizado?
* risco de sessão quebrada?

9. BANCO DE DADOS

* tenants

* logs

10. SEGURANÇA GERAL
