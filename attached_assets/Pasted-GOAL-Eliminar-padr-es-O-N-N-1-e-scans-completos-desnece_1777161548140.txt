GOAL:
Eliminar padrões O(N), N+1 e scans completos desnecessários no domínio,
SEM alterar comportamento, SEM alterar response shape e SEM alterar regras de negócio.

STRICT RULES:
- NÃO alterar nenhum endpoint
- NÃO alterar nenhum response (shape, status, mensagens)
- NÃO alterar lógica de negócio
- NÃO alterar validações
- NÃO alterar contratos do service
- NÃO alterar nomes de campos retornados
- NÃO otimizar mudando regra (apenas acesso a dados)
- Se houver qualquer risco de mudança de comportamento → PARAR e reportar

---

STEP 1 — DISCOVERY (PERFORMANCE HOTSPOTS)

Escanear:

server/modules/**/services
server/modules/**/repositories

Identificar padrões:

1. FULL SCAN:
- storage.getProducts()
- storage.getOrders()
- storage.getInvoices()
seguido de .find(...) ou .filter(...)

2. N+1:
- loops com await dentro:
  for (...) {
    await repo.get(...)
  }

3. LOOKUP FREQUENTE:
- checkDuplicate
- checkCode
- findById feito manualmente

---

STEP 2 — DEFINIR MÉTODOS NOVOS NO STORAGE

No arquivo:

server/services/storage.ts

Adicionar métodos (sem remover os existentes):

// PRODUCTS
getProductById(id: number)
findProductByCode(code: string)

// ORDERS
getOrderById(id: number)
orderHasProduct(orderId: number, productId: number)

// INVENTORY (se aplicável)
getInventorySettingById(id: number)

⚠️ IMPORTANTE:
- Implementação deve usar fonte real (DB ou memória existente)
- NÃO alterar retorno dos métodos antigos
- NÃO remover getProducts()

---

STEP 3 — IMPLEMENTAÇÃO SEGURA

Substituir APENAS acesso a dados:

ANTES:
const products = await storage.getProducts();
const product = products.find(p => p.id === id);

DEPOIS:
const product = await storage.getProductById(id);

---

ANTES:
const exists = products.some(p => p.code === code);

DEPOIS:
const exists = await storage.findProductByCode(code);

---

ANTES (N+1):
for (const order of orders) {
  const detail = await repo.getOrder(order.id);
}

DEPOIS:
✔️ manter lógica
✔️ mas usar método direto:
await storage.getOrderById(order.id)

(ou otimizar para lookup direto se possível)
