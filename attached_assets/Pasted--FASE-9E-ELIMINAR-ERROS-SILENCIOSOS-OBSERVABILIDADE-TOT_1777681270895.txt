## 🛠️ ETAPA 1 — PADRÃO GLOBAL

Para TODO catch que:

- está vazio
- usa apenas console.log / console.error
- ou ignora erro

Aplicar o padrão:

ts catch (err: any) {   logSecurity(     `[NOME_EVENTO] context=${...} | error=${err?.message ?? "unknown"}`   ); } 

---

## 🛠️ ETAPA 2 — NOMES DOS EVENTOS (PADRÃO)

Usar nomes claros e rastreáveis:

| Contexto | Evento |
|--------|--------|
| logística dispatch | LOGISTICS_DISPATCH_FAILED |
| NF-e alert log | NFE_ALERT_LOG_FAILED |
| faturamento cron | BILLING_CRON_FAILED |
| email scheduler | EMAIL_SCHEDULER_TICK_FAILED |
| alerts service | ALERT_DELIVERY_FAILED |
| storage/log persist | STORAGE_WRITE_FAILED |
| push notification | PUSH_NOTIFICATION_FAILED |
| backup automático | BACKUP_FAILED |
| seed/init | SYSTEM_SEED_FAILED |

## 🛠️ ETAPA 3 — CONTEXTO MÍNIMO

Sempre incluir no log (quando disponível):

- orderId
- companyId
- userId
- jobName / service
- requestId (se existir no escopo)

Exemplo:

ts logSecurity(   `[EMAIL_SCHEDULER_TICK_FAILED] job=emailScheduler | error=${err?.message}` );
