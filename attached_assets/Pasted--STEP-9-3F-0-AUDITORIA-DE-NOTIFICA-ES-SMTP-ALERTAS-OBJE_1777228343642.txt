## STEP 1 — MAPEAR SMTP EXISTENTE

### Procurar no projeto:

- "nodemailer"
- "smtp"
- "mail"
- "transporter"
- "sendMail"
- ".env"

### Identificar:

- Onde o transporter é criado
- Se já existe serviço tipo:
  - email.service.ts
  - mailer.ts
- Se já existem templates

### Retornar:

json id="smtp_audit" {   "smtpExiste": true/false,   "arquivo": "caminho do arquivo",   "reutilizavel": true/false,   "observacoes": "ex: já usado para pedidos / login / reset senha" } 

---

## STEP 2 — MAPEAR SISTEMA DE NOTIFICAÇÕES

### Procurar:

- console.error / console.warn com padrão
- logs estruturados
- qualquer tentativa de alerta

### Verificar:

- Já existe algo como:
  - alert.service
  - notification.service
  - webhook

### Retornar:

json id="notif_audit" {   "existeSistema": true/false,   "tipo": "email | log | nenhum",   "reutilizavel": true/false } 

---

## STEP 3 — MAPEAR USERS + ROLES

### Identificar:

- Onde está o user:
  - useAuth()
  - req.user
  - session

### Descobrir:

- Quais roles existem exatamente (strings reais)
- Exemplo esperado:
  - ADMIN
  - MASTER
  - DIRECTOR

### Retornar:

json id="roles_audit" {   "rolesEncontradas": ["ADMIN", "MASTER", "DIRECTOR"],   "campoRole": "user.role",   "caseSensitive": true/false } 

---

## STEP 4 — MAPEAR ONDE GUARDAR WHATSAPP

### Procurar no schema:

- tabela users
- tabela companies
- tabela settings/config

### Verificar:

- existe campo JSON?
- existe tabela de configurações?

### Objetivo:

Definir melhor lugar para:

ts whatsapp_alert_numbers: string[] 

### Retornar:

json id="whatsapp_storage" {   "localRecomendado": "users | settings | nova tabela",   "justificativa": "motivo técnico" } 

---

## STEP 5 — SEGURANÇA (CRÍTICO)

Definir regra:

Somente roles podem gerenciar alertas:

- ADMIN
- MASTER
- DIRECTOR

### Validar:

- Onde essa verificação já existe no projeto
- Se há middleware ou helper reutilizável

### Retornar:

json id="security_audit" {   "temMiddleware": true/false,   "comoValidar": "ex: req.user.role === 'ADMIN'",   "reutilizar": true/false }
