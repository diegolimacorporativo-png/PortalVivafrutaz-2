- Grupo CORE (sempre recebe tudo)
- Grupos por categoria (recebem apenas o que importa)

❌ NÃO duplicar persistência  

---

# 🧠 CONCEITO

## 🔒 CORE (fixo — SEMPRE recebe)
- DIRECTOR  
- ADMIN  
- MASTER  
- DEV  

👉 Recebem TODOS os alertas

---

## ⚙️ CATEGORIA (opcional)
Baseado em:

ts context.category 

Exemplos:
- FINANCE
- TECH
- OPERATIONS

---

# 🧱 ARQUITETURA

## Novo arquivo:
server/services/alerts.routing.ts

---

## Estrutura:

ts export const CORE_ROLES = ["DIRECTOR", "ADMIN", "MASTER", "DEV"];  export const CATEGORY_ROUTING = {   FINANCE: ["FINANCE"],   TECH: ["DEV", "ADMIN"],   OPERATIONS: ["ADMIN"], }; 

---

## Função principal:

ts resolveRecipients(input) 

Responsável por:

1. Iniciar com CORE_ROLES  
2. Ler input.context?.category  
3. Se existir:
   - adicionar roles da categoria  
4. Remover duplicados  
5. Retornar lista final de roles  

---

# 🔌 INTEGRAÇÃO

## Onde aplicar:

👉 SOMENTE dentro do fluxo de envio

### No emitAlertSmart (SEM alterar lógica interna)

ANTES de enviar:

- chamar resolveRecipients()
- passar resultado como:

ts recipientsRoles: [...]

ts emitAlertSmart({   severity,   title,   message,   context: {     ...,     category: "FINANCE"   },   recipientsRoles: ["DIRECTOR", "ADMIN", "DEV", "MASTER", "FINANCE"] });
