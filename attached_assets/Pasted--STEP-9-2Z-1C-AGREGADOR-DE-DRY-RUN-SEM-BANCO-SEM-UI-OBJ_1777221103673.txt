## STEP 9.2Z.1C — AGREGADOR DE DRY-RUN (SEM BANCO, SEM UI)

### OBJETIVO

Capturar métricas de impacto do faturamento:

- Quantos pedidos seriam bloqueados
- Por tipo (semanal/mensal)
- Últimas ocorrências

SEM usar banco de dados

---

## 🚫 REGRAS

- NÃO criar tabela
- NÃO usar storage
- NÃO criar dashboard complexo
- Tudo em memória (temporário)

---

## STEP 1 — CRIAR STORE EM MEMÓRIA

Arquivo:

server/modules/nfe/dryrun-metrics.ts

ts id="3k92jd" type DryRunEvent = {   orderId: number;   tipo: string;   motivo: string;   at: number; };  const MAX_EVENTS = 200;  const metrics = {   total: 0,   byTipo: {} as Record<string, number>,   events: [] as DryRunEvent[], };  export function recordDryRun(event: DryRunEvent) {   metrics.total++;    metrics.byTipo[event.tipo] = (metrics.byTipo[event.tipo] || 0) + 1;    metrics.events.unshift(event);    if (metrics.events.length > MAX_EVENTS) {     metrics.events.pop();   } }  export function getDryRunMetrics() {   return metrics; }
