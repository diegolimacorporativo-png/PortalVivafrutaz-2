STEP 1 — FIX W1 (AUTH CONTROLLER SESSION LOG)

Arquivo:
server/modules/auth/auth.controller.ts

Problema:
Existe log dentro de callback de sessão:

console.error("[LOGIN] Erro ao salvar sessão:", err)

Solução:
- Incluir requestId no log
- Sem mudar fluxo

ANTES:
console.error("[LOGIN] Erro ao salvar sessão:", err)

DEPOIS:
console.error(
  `[${req.requestId}] [auth.controller] session save failed`,
  err
)

⚠️ IMPORTANTE:
- NÃO alterar req.session.save()
- NÃO transformar em async/await
- NÃO mover o código
- Apenas enriquecer log

---

STEP 2 — FIX W2 (INVENTORY CONTROLLER SILENT CATCHES)

Arquivo:
server/modules/inventory/inventory.controller.ts

Problema:
Alguns blocos:

} catch (e) {
  if (e instanceof AppError) {
    return res.status(...).json(...)
  }
  throw e;
}

→ NÃO logam erro operacional (AppError)

Solução:
- Adicionar log ANTES do return

DEPOIS:

} catch (e) {
  if (e instanceof AppError) {
    console.warn(
      `[${req.requestId}] [inventory.controller] <method> failed`,
      e
    );
    return res.status(...).json(...);
  }
  throw e;
}

⚠️ IMPORTANTE:
- NÃO alterar resposta
- NÃO alterar status
- NÃO alterar mensagens
- NÃO remover throw e
- Apenas adicionar log

Aplicar em TODOS os catches desse padrão no arquivo

---

STEP 3 — FIX W3 (EXPRESS LOGGER SEM requestId)

Arquivo:
server/app.ts

Trecho atual:
logger existente:

console.log(
  `[express] ${method} ${path} ${status} in ${duration}ms ::`,
  body
);

Problema:
Não inclui requestId

Solução:
Prefixar com requestId

DEPOIS:

console.log(
  `[${req.requestId}] [express] ${method} ${path} ${status} in ${duration}ms ::`,
  body
);

⚠️ IMPORTANTE:
- NÃO alterar estrutura do log
- NÃO remover body
- NÃO mudar formato além do prefixo
- NÃO remover esse logger
