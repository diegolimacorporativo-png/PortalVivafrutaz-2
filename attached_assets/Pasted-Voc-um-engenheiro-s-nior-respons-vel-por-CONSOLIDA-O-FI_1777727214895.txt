🔧 1. CRIAR FLUXO ÚNICO DE DECISÃO (OBRIGATÓRIO)
━━━━━━━━━━━━━━━━━━━━━━━

Refatorar lógica de autenticação para seguir EXATAMENTE esta ordem única:

1. SESSION VALIDATION
   - tokenVersion check
   - session existence
   - deviceId validation

2. INFRASTRUCTURE PROTECTION (L1)
   - apenas detecção de burst / abuso
   - NÃO pode bloquear login sozinho
   - apenas sinaliza risco e registra evento

3. BUSINESS RATE LIMIT (L2 - DB)
   - auth_attempts lookup
   - decisão final de bloqueio (allowed / denied)

4. ACCOUNT STATE CHECK
   - isLocked
   - mustChangePassword
   - active status

5. CREDENTIAL VALIDATION
   - bcrypt compare

6. FINAL RESULT HANDLING
   - success → reset counters + logAuthEvent(LOGIN_SUCCESS)
   - failure → recordAttempt + L1 increment
