## STEP 2 — CRIAR ALERT SERVICE

Arquivo:
server/services/alerts.service.ts

FUNÇÃO PRINCIPAL:

emitAlert({
  severity: "ALERT" | "CRITICAL",
  title: string,
  message: string,
  context?: any
})

---

### IMPLEMENTAÇÃO

1. Buscar destinatários via system_settings
2. Filtrar enabled === true
3. Para cada canal:

EMAIL:
- usar mailer.ts (sendMail)
- assunto: [CRON ALERT] ou [CRON CRITICAL]

SLACK:
- POST webhook (fetch)
- payload:
{
  text: "CRON ALERT\nErro detectado\njson\n{...}\n"
}

WHATSAPP:
- NÃO implementar envio ainda
- apenas deixar preparado (console.log "[WHATSAPP_PENDING]")

---

## STEP 3 — RATE LIMIT (ANTI-SPAM)

No alerts.service:

Criar cache em memória:

const lastSent = new Map<string, number>();

Chave:
"cron:alert" ou "cron:critical"

Regra:
- bloquear envio se último envio < 10 minutos

## STEP 4 — INTEGRAR COM CRON

Arquivo:
server/jobs/faturamento.cron.ts

ADICIONAR:

import { emitAlert } from "@/services/alerts.service";

---

### ON ERROR:

if (errors > 0) {
  await emitAlert({
    severity: "ALERT",
    title: "Erros no cron de faturamento",
    message: Foram detectados ${errors} erros,
    context: { total, errors }
  });
}

---

### ON CRITICAL:

if (success === 0 && total > 0) {
  await emitAlert({
    severity: "CRITICAL",
    title: "Falha total no cron",
    message: "Nenhuma NF foi emitida",
    context: { total }
  });
}
